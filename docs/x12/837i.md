# 837I — Institutional Claims

The 837I (Institutional) transaction is used to submit healthcare claims for facility and institutional services (hospitals, skilled nursing facilities, etc.). The parser supports 837I for inbound processing. Outbound 837I mapping follows the same pattern as 837P — implemented as a project-specific activity in the config project.

## Differences from 837P

| Aspect | 837P (Professional) | 837I (Institutional) |
|---|---|---|
| Claim level | SV1 (professional services) | SV2 (institutional services) |
| Revenue codes | Not used | Required per service line |
| UB-04 fields | Not applicable | Admission type, source, status |
| Billing provider | Individual or group NPI | Facility NPI |
